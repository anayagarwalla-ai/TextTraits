TextTraits Free Deploy Requirements For The Repository Owner
============================================================

This is the free-only checklist for the owner of csboi/TextTraits to deploy TextTraits as a hosted beta. The merged production-ready app contains the app code, Dockerfile, Procfile, render.yaml, WSGI entrypoint, Postgres support, production preflight checks, and deploy docs.

1. Merge or deploy the correct branch
   - Deploy main only after the production-readiness merge commit has been pushed to GitHub.
   - If main has not been advanced to that merge commit yet, deploy codex/production-readiness instead.
   - Do not deploy an older branch unless these production-readiness changes are included.

2. Create a free dynamic web service
   - Use a free web service host such as Render, Railway, Fly.io, or another free Python/Docker host.
   - Select this repository as the source.
   - Deploy it as a dynamic web service, not a static site.
   - If the host detects the Dockerfile, use Docker.
   - If the host asks for a start command, use:
     gunicorn texttraits_app.wsgi:application --bind 0.0.0.0:$PORT --workers 2 --timeout 60

3. Create a free hosted Postgres database
   - Use a free hosted Postgres provider such as Supabase, Neon, or a free Render Postgres instance if available.
   - Copy the Postgres URI connection string.
   - Use the provider's SSL-capable connection string.
   - Keep the database password private.

4. Set production environment variables
   Set these in the hosting provider's environment/secret settings:
   - TEXTTRAITS_ENV=production
   - TEXTTRAITS_SECRET_KEY=<generated high-entropy secret key>
   - TEXTTRAITS_PUBLIC_BASE_URL=<final hosted HTTPS URL>
   - TEXTTRAITS_SECURE_COOKIES=true
   - TEXTTRAITS_TRUST_PROXY=true
   - TEXTTRAITS_DB_SSLMODE=require
   - DATABASE_URL=<Supabase pooler URL for project olacwelhrgzxzrxtmvto>
   - ENABLE_DEV_TOOLS=false

   Supabase CLI setup for this project:
   - supabase login
   - supabase init
   - supabase link --project-ref olacwelhrgzxzrxtmvto

   Or run:
   - scripts/setup_supabase_cli.sh

5. Add free transactional email
   Account verification and password reset need real email delivery.
   Free options include Brevo, SendGrid free tier, or SMTP2GO free tier.
   Set:
   - TEXTTRAITS_EMAIL_PROVIDER=smtp
   - TEXTTRAITS_FROM_EMAIL=<verified sender email>
   - TEXTTRAITS_SMTP_HOST=<provider host>
   - TEXTTRAITS_SMTP_PORT=587
   - TEXTTRAITS_SMTP_USERNAME=<provider username/API key>
   - TEXTTRAITS_SMTP_PASSWORD=<provider password/API key>
   - TEXTTRAITS_SMTP_TLS=true

6. Verify the sender email or sender domain
   - Complete sender verification inside the free email provider.
   - This reduces blocked or spammed password reset and verification emails.
   - A paid custom domain is not required for a beta if the provider supports verified sender email.

7. Add free error monitoring
   - Create a free Sentry project.
   - Copy the Sentry DSN.
   - Set SENTRY_DSN in the hosting provider.
   - This is optional for a private demo, but strongly recommended before a public beta.

8. Use the host's free HTTPS URL
   - Use the free HTTPS URL from the hosting provider, such as a render.com URL.
   - Set that exact URL as TEXTTRAITS_PUBLIC_BASE_URL.
   - A paid custom domain is not required.

9. Review privacy and terms before launch
   - Review the included privacy and terms routes plus LEGAL_REVIEW.md.
   - Approve the wording for the actual beta behavior.
   - Formal legal review can wait for a small private beta, but public launch should get real review.

10. Run pre-deploy validation
   Run these after environment variables are configured:
   - python3 scripts/preflight.py
   - python3 tests/smoke_test.py
   - python3 tests/browser_smoke_test.py
   - python3 tests/accessibility_smoke_test.py
   - python3 tests/production_smoke_test.py
   - python3 tests/security_regression_test.py
   - python3 tests/config_regression_test.py
   - python3 tests/frontend_regression_test.py
   - python3 tests/env_storage_regression_test.py

11. Confirm the live health endpoint
   After deploy, open:
   - https://<live-app-url>/health
   Confirm:
   - ok is true
   The public health endpoint intentionally exposes only aggregate readiness. Use deploy logs, Sentry, database provider status, and preflight output to verify Postgres/email/integration details.

Can Wait Until After Free Beta
==============================

- Paid custom domain
- Paid database tier
- Paid email tier
- Real HubSpot/Salesforce/Gmail/Outlook OAuth credentials
- Chrome extension packaging
- Enterprise SSO
- Full legal counsel review
- Advanced backups beyond the free database provider's defaults
